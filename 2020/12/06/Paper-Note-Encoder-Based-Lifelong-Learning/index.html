Encoder Based Lifelong Learning

Amal Rannen, Rahaf Aljundi, Matthew B. Blaschko, Tinne Tuytelaars

ICCV 2017

Paper Adress

Background

和 Learning without Forgetting (LwF) 那篇一样，先拿以下三种老方法祭旗开路：

**Feature extraction：**在遇到新任务时，模型参数不改变，相当于只是对新任务的数据做表征，做特征提取。对旧任务友好，对新任务不友好。

**Fine-tuning：**遇到新任务时，模型在旧任务的基础上做fine-tune，容易学了后面忘了前面。对旧任务不友好，对新任务友好。 

**Joint training：**把所有任务的数据汇到一起，一块训练。这显然是不现实的，不然也不需要持续学习了。提到 Joint training 的意义也仅在于把它作为持续学习的理论上界而已。

对于本文而言，比较关键的两篇工作：LwF [14] 和 EWC [12]

Learning without Forgetting (LwF): proposes a way to train convolutional networks, where a shared model is used for the different tasks but with separate classification layers. When a new task is presented, a new classification layer is added. Then, the model is fine-tuned on the data of the new task, with an additional loss that incorporates the knowledge of the old tasks. This loss tries to keep the previous task predictions on the new data unchanged. Such a solution reduces the forgetting but is heavily relying on the new task data. As a consequence, it suffers from a build-up of errors when facing a sequence of tasks [1].

LwF proposes to preserve the previous performance through the knowledge distillation loss introduced in [11]. They consider a shared convolutional network between the different tasks in which only the last classification layer is task specific. When encountering a new task, the outputs of the existing classification layers given the new task data are recorded. During training, these outputs are preserved through a modified cross-entropy loss that softens the class probabilities in order to give a higher weight to the small outputs. This method reduces the forgetting, especially
when the datasets come from related manifolds. Nevertheless, it has been shown by [22] that LwF suffers from a build up of errors in a sequential scenario where the data comes from the same environment. Similarly, [1] shows that the LwF performance drops when the model is exposed to a sequence of tasks drawn from different distributions.

Elastic Weight Consolidation (EWC): Rather than having a data-oriented analysis, they consider the knowledge gained in the model itself, and transfer it from one task to another in a Bayesian update fashion. The method relies on approximating the weight distribution after training the model on the new task. A Gaussian distribution, for which the mean is given by the optimal weights for the first task, and the variance given by the diagonal of the Fisher information matrix is used as an approximation. Such a solution is based on a strong principle and gives interesting results. However, it requires to store a number of parameters that is comparable to the size of the model itself.

EWC is to constrain the weights WiW_iWi​ while training for a second task with an additional loss ∑iλ2Fi(Wi−W1,i∗)\sum_i\cfrac{\lambda}{2}F_i(W_i-W_{1,i}^*)∑i​2λ​Fi​(Wi​−W1,i∗​) where W1,i∗W_{1,i}^*W1,i∗​ are the optimal weights of the first task, and FiF_iFi​ the diagonal terms of their Fisher information matrix. The use of the Fisher matrix prevents the weights that are important for the first task to change much. In our point of view, this method, despite its success, has two drawbacks. First, the method keeps the weights in a neighborhood of one possible minimizer of the empirical risk of the first task. However, there could be another solution that can give a better compromise between the two tasks.（这个问题提得漂亮！） Second, it needs to store a large number of parameters that grows with the total number of weights and the number of tasks. For these reasons, rather than constraining the weights, we choose to constrain the resulting features, enforcing that those that are important for the previous tasks do not change much. By constraining only a sub-manifold of the features, we allow the weights to adjust so as to optimize the features for the new task, while preserving those that ensure a good performance on the previous tasks.

Main Idea

We introduce the use of autoencoders as a tool to preserve the knowledge from one task while learning another. For each task, an undercomplete autoencoder is trained after training the task model. It captures the most important features for the task objective. When facing a new task, this autoencoder is used to ensure the preservation of those important features. This is achieved by defining a loss on the reconstructions made by the autoencoder. In this manner, we only restrict a subset of the features to be unchanged while we give the model the freedom to adapt itself to the new task using the remaining capacity. Shortcomings of Learning without Forgetting

E(X(2),Y(2))[l(T1∘T∘F(X(2)),T1∗∘T∗∘F∗(X(2)))+l(T2∘T∘F(X(2)),Y(2))]\mathbb{E}_{(\mathcal{X}^{(2)},\mathcal{Y}^{(2)})} [l(T_1\circ T\circ F(\mathcal{X^{(2)}}),T^*_1\circ T^*\circ F^*(\mathcal{X^{(2)}}))+l(T_2\circ T\circ F(\mathcal{X^{(2)}}),\mathcal{Y}^{(2)})] E(X(2),Y(2))​[l(T1​∘T∘F(X(2)),T1∗​∘T∗∘F∗(X(2)))+l(T2​∘T∘F(X(2)),Y(2))]

那我们看上面这个式子，第一项其实是要求在训练任务二时，新模型与旧模型的“表征”和“分类”能力尽可能是相似的；第二项是在要求在第二个任务上有尽可能好的表现。那如果要让这个式子尽可能小，那实质上是在要求任务一与任务二的数据分布（即  X(1)\mathcal{X}^{(1)}X(1) 与 X(2)\mathcal{X}^{(2)}X(2)）尽可能相似一些，否则 LwF 的表现就不会太好。也就是说，LwF 有效的前提是各个任务之间的数据分布不能相差太远。

Informative feature preservation

论文的想法是学习得到一个关于特征空间 F(X(1))F(\mathcal{X}^{(1)})F(X(1)) 的 submanifold，这个 submanifold 应该包含了任务一的大多数特征。在训练任务二的过程中，特征 F(X(2))F(\mathcal{X}^{(2)})F(X(2)) 在这个 submanifold 上的投影不应该改变太多，这样有两个好处：1) F(X(2))F(\mathcal{X}^{(2)})F(X(2)) 在任务一上仍然是 informative 的；2) 只要F(X(2))F(\mathcal{X}^{(2)})F(X(2)) 在 submanifold 上的投影不改变，它在任务二仍然有继续做适应性改变的空间。（这个思路就很像EWC了。）如 Figure 2 所示。
