Scalable and Order-robust Continual Learning with Additive Parameter Decomposition

Jaehong Yoon, Saehoon Kim, Eunho Yang, Sung Ju Hwang

ICLR 2020

[paper] [code]

Background

文章首先指出了之前的持续学习方法的一些问题：(1) 没有验证过在足够大量的任务场景下的模型可扩展性，包括避免灾难性遗忘的效果和存储、计算开销的高效性。(2) 文章提出了一个对于持续学习算法的新的评价指标：任务顺序敏感性（task order sensitivity）。如果考虑到任务之间的公平性（比如：疾病诊断）的时候，这个指标就显得很重要了。但是之前的持续学习方法对这个指标的讨论也很少。

Main Idea

为处理以上两个问题，我们提出了一个伴有 Additive Parameter Decomposition (APD) 的新持续学习模型。APD 将网络每一层的参数分解为两种类型：task-shared 和 task-specific 。我们将带有APD的网络称为 APD-net。当有新的任务到来，模型首先尽可能利用 task-shared 参数，然后使用很稀疏的 task-adaptive 的参数来对共享参数学习不了的部分进行学习。在此基础上，由于使用唯一的一组共享参数对于各个不同分布的任务而言，可能过于单薄，我们将 task-adaptive 参数进行了聚类，获得了一种层级化的共享参数。如 Figure 2 所示。

Figure 2. 对于APD的图解。σ\boldsymbol{\sigma}σ 表示共享参数，τt\boldsymbol{\tau_t}τt​ 表示 task-adaptive 参数，后续的任务只会去更新 σ\boldsymbol{\sigma}σ ，而不会影响之前的 τt\boldsymbol{\tau_t}τt​。Mt\mathcal{M}_tMt​ 是 σ\boldsymbol{\sigma}σ 上的 task-adaptive mask，以此来获得与特定任务相关的知识。τt\boldsymbol{\tau_t}τt​ 的稀疏性和层级化的知识融合大大增强了可扩展性。

Method

Additive Parameter Decomposition

前文已经说过，为了在减缓灾难性遗忘的同时，尽可能地减少扩展网络所引入的参数量，我们将网络参数 θ\boldsymbol{\theta}θ 分解为 task-shared 参数矩阵 σ\boldsymbol{\sigma}σ 和 task-adaptive 矩阵 τ\boldsymbol{\tau}τ ，亦即：对于任务 ttt 有， θt=σ⊗Mt+τt\boldsymbol{\theta}_t=\boldsymbol{\sigma}\otimes\mathcal{M}_t+\boldsymbol{\tau}_tθt​=σ⊗Mt​+τt​ 。目标函数如下所示：

分析一下这个目标函数：(1) 先看第一个正则项，是为了使 τt\boldsymbol{\tau}_tτt​ 尽可能地稀疏。(2) 第二个正则项是说，σ\boldsymbol{\sigma}σ 可以更新，但是希望相比于遇到新任务之前的共享参数 σ(t−1)\boldsymbol{\sigma}^{(t-1)}σ(t−1) 而言变化不要太大，当然除了 l2−norml_2-norml2​−norm 也可以使用其它类型的正则，比如说 EWC 所使用的那种。(3) masking variable Mt\mathcal{M}_tMt​ 是伴有可学习参数 vt\boldsymbol{\rm{v}_t}vt​ 的 sigmoid 函数，应用在了 σ\boldsymbol{\sigma}σ 每一层的输出通道（output channels）或神经元上。

现在可以大概感受到为什么说这个方法是对于任务顺序健壮（order-robust）的了，因为语义漂移（semantic-drift）只可能出现在对应于通用知识的 task-shared 参数上，而不会影响到之前任务的 task-specific 参数。

Order Robust Continual Learning with Retroactive Parameter Updates

如上一节所言，在共享参数上仍然会出现语义漂移问题，这依然会导致对任务顺序敏感的模型。为了使模型对任务顺序更加健壮，我们添加了另外一项正则来避免参数级的 drift，而不需要对之前的任务做显式的训练。首先来看一下新的目标函数：

min⁡σ,τ1:t,v1:t L({σ⊗Mt+τt};Dt)+λ1∑i=1t∥τi∥1+λ2∑i=1t−1∥θi∗−(σ⊗Mi+τi)∥22(2)\min\limits_{\boldsymbol{\sigma},\boldsymbol{\tau}_{1:t},\boldsymbol{\rm{v}}_{1:t}}\ \mathcal{L}(\{\boldsymbol{\sigma}\otimes\mathcal{M}_t+\boldsymbol{\tau}_t\};\mathcal{D}_t) +\lambda_1\sum_{i=1}^t\|\boldsymbol{\tau}_i\|_1 +\lambda_2\sum_{i=1}^{t-1}\|\boldsymbol{\theta}_i^*-(\boldsymbol{\sigma}\otimes\mathcal{M}_i+\boldsymbol{\tau}_i)\|_2^2\quad(2) σ,τ1:t​,v1:t​min​ L({σ⊗Mt​+τt​};Dt​)+λ1​i=1∑t​∥τi​∥1​+λ2​i=1∑t−1​∥θi∗​−(σ⊗Mi​+τi​)∥22​(2)

Selective task forgetting

在实际场景中，前面所学到的一些任务可能渐渐地不再需要，那我们就希望模型能把这些任务“忘记”，为学习后续任务保留空间。对于APD而言，可以通过 drop out 掉 task-adaptive 参数 τt\boldsymbol{\tau}_tτt​ 来忘记掉任务 ttt。这样也就不会影响到其余任务的 task-adaptive 参数了。

Discussion

文章中有一句话 hit 到我了：""Even with well-defined regularizers, it is nearly impossible to completely avoid catastrophic forgetting, since in practice, the model may encounter an unlimited number of tasks. 我比较认同这句话，想要在一个固定大小的网络中学习很多很多的任务是不现实的，除非这个网络本身已经足够大了。比较好的思路是如何去高效地扩展网络，比如说先扩展再剪枝？根据某些启发式的或者什么正则化的方式去克制地进行扩展？

这篇文章也是引入了太多超参数，我发现这位一作的风格就是引入好多超参来构造一个很复杂的模型。这样的模型实用性到底有多高呢？
