Continual Learning of a Mixed Sequence of Similar and Dissimilar Tasks

Zixuan Ke, Bing Liu, Xingchang Huang

NeurlIPS 2020

[paper] [code]

Background

作者认为，理想情况下的持续学习在学习新任务 t 时应该满足以下四点：(1) 避免灾难性遗忘： 不忘记之前所学的任务；(2) 知识的前向迁移： 如果新任务 t 与之前的某些任务很相似，有可以共享的知识，应该将之前所学习到的知识向前迁移，帮助对任务 t 的学习；(3) 知识的后向迁移： 新学习到的知识也能向后迁移，帮助之前所学习的与之相似的任务；(4) 能够学习一系列混合任务——既有相似的任务，也有不相似的任务，在这种情况下依然能够同时满足 (1)、(2)、(3)。目前还没有文章能够同时做到以上四点。这篇文章由此粉墨登场。

这篇文章想要在 task continual learning (TCL, also known as task incremental learning) 的setting 下解决这个问题。TCL 与 class continual learning (CCL, also known as class incremental learning) 是两个相对的概念。

Main Idea

本文提出了一个新的 TCL 模型 CAT (Continual learning with forgetting Avoidance and knowledge Transfer)，该模型可以有效地学习一系列混合的相似和不相似的任务，并且达到之前的四个目标。CAT 使用一个 knowledge base (KB) 来保存此前学习过程中所得到的知识，并且所有任务都可以共享它。在学习一个新任务 t 之前，learner 首先自动地识别此前学习过程中遇到的与任务 t 相似的任务 TsimT_{sim}Tsim​。其余与 t 不相似的任务被标识为 TdisT_{dis}Tdis​。在学习 ttt 的过程中，learner 使用 task masks ( TM ) 来保护与 t 不相似的任务（即 TdisT_{dis}Tdis​ 中的任务）的知识，也就是说对这些任务重要的参数不会受到影响（no forgetting）。TM是从之前的任务中学习而来的。对于任务 t，也会学习得到 masks（一层神经网络对应一个mask）来在将来保护它的知识。对于 TsimT_{sim}Tsim​ 中与 t 相似的任务，learner会学习得到一个 knowledge transfer attention (KTA)，以此选择对 t 有用的知识从 TsimT_{sim}Tsim​ 迁移到新任务，来改善新任务的学习（forward knowledge transfer）。在学习任务 t 的过程中，CAT 也允许 TsimT_{sim}Tsim​ 中的任务的知识更新，以使得其中的某些任务的表现也能得到提升（backward knowledge transfer）。

Mehtod

CAT 模型如 Figure 1 (A) 所示。在该图底部，分别是输入数据和任务 ID ttt。在它上面是 knowledge base (KB)，KB 可以是任意的可微的层（CAT 是以两层的全连接层和一个CNN结构进行实验）。任务ID ttt 可以生成三种不同的任务ID embeddings。其中的两个是与KB中相应层的输出进行 element-wise 的相乘，而最后一个是知识迁移模块（knowledge transfer module）的输入（在图的右上部分）。KB的输出通向两个分支（蓝色和红色的矩形块）。蓝色分支是总是存在的，该分支将学习一个任务 ttt 的分类模型 fmaskf_{mask}fmask​，并同时会为KB中的每一层学习一个binary mask（称为 task mask ( TM )），用于指示在该层中对于任务重要/有用的 units。该 mask 用于在学习将来任务时保护所习得的任务 ttt 的知识。如果任务 ttt 与先前的某些任务相似，则 KB 的输出也将通向右侧分支以进行选择性的知识迁移，这是通过 knowledge transfer attention (KTA) 实现的。在KTA之上，为任务 ttt 构建了另一个分类器 fKTAf_{KTA}fKTA​，它利用迁移的知识来学习 ttt。在测试时，此分类器与 fmaskf_{mask}fmask​ 应该择一使用。fmaskf_{mask}fmask​ 仅在先前没有与任务 ttt 相似的任务时才用于测试。请注意，在 task continual learning (TCL) 中，需要使用任务ID，因为在测试时每个测试实例将通过其相应任务的分类器/模型对其进行测试。

Figure 1：（B）是 task masking 和知识迁移的示意图。关于（B）的一些需要注意的要点：在训练之前的矩阵（Before Training）中，那些值为0的单元格是要被保护（masked）的 units，那些值为1的单元格是要共享的 units，其任务与当前任务相似。那些没有数字的单元格是空闲units（即还未被使用）。在训练后的矩阵（After Training）中，那些值为1的单元格表示对于当前任务很重要的 units，这些单元格将来会被用作 masks。其余的单元格对于该任务则并不重要。那些没有颜色的0值单元格没有被任何任务使用。

Preventing Forgetting for Dissimilar Tasks: Task Masks

把新任务 t 之前的所有任务的集合记为 T\mathcal{T}T 。把与任务 t 相似的任务的集合记作 Tsim⊆T\mathcal{T}_{sim}\subseteq\mathcal{T}Tsim​⊆T ，与之不相似的任务的集合记作 Tdis=T−Tsim\mathcal{T}_{dis}=\mathcal{T}-\mathcal{T}_{sim}Tdis​=T−Tsim​ 。在学习任务 t 的过程中，为了防止 Tdis\mathcal{T}_{dis}Tdis​ 中不相似任务的遗忘，我们识别出KB中被这些任务使用的 units，并阻塞掉流经这些 units 的梯度（即：置它们的梯度为 0）。为了实现此目标，在训练任务 ttt 的分类器/模型时，为每个任务 ttt 、KB 的每一层训练一个 task mask (a binary mask) ml(t)m_l^{(t)}ml(t)​ ，用于指示对于该任务而言，该层中哪些 units 是重要的。 这里我们借鉴了 hard attention 的想法（Serrà et al., 2018），利用 task ID embedding 来训练任务。

Knowledge Transfer from Similar Tasks: Knowledge Transfer Attention

对于新任务 ttt ，如果之前有相似的任务（Tsim≠∅\mathcal{T}_{sim}\neq\emptyTsim​​=∅），我们希望从 Tsim\mathcal{T}_{sim}Tsim​ 进行知识迁移，以使得 ttt 学习得更好。由于实际上，每个任务都有不适合于其它任务的而是其专有的知识，所以这种知识迁移应该是有选择性的。我们为此提出了 knowledge transfer attention (KTA)。想法是通过注意力机制给予之前不同的任务以不同的重要性，这样就能使之前任务的知识可以选择性地迁移到新的任务。之前任务的参数也是可更新的，能够自动实现 backward knowledge transfer ，所以在训练新任务 ttt 的时候也可以改善之前的任务模型。

我们可以通过读取 {ml(j)}\{m_l^{(j)}\}{ml(j)​} 知道各个 units 是对应于哪个任务 jjj。对于 Tsim\mathcal{T}_{sim}Tsim​ 中与当前任务 ttt 相似的任务 isimi_{sim}isim​ ，我们通过对KB使用 ml(isim)m_l^{(i_{sim})}ml(isim​)​ 可以计算它的 masked KB output hmask(isim)h_{mask}^{(i_{sim})}hmask(isim​)​ （最后一层）。

Task Similarity Detection

在以上讨论中，我们假设我们知道新任务 ttt 的一组与之相似的先前的任务 Tsim\mathcal{T}_{sim}Tsim​。现在，我们介绍如何为给定任务 ttt 找到相似任务。我们使用二值向量TSV（任务相似性向量）来指示每个先前的任务 k∈Tk\in\mathcal{T}k∈T 是否与新任务 ttt 相似。

我们通过确定从先前任务 kkk 到当前任务 ttt 是否存在 positive knowledge transfer 来定义任务相似性。迁移模型（ transfer model ）fk→tf_{k\rarr t}fk→t​ 用于从先前任务 kkk 到当前任务 ttt 迁移知识。单独的任务模型 f∅f_{\varnothing}f∅​ （称作 参照模型，reference model ）被用于独立学习任务 ttt。如果下面的 statistical risk 成立，表明这是positive knowledge transfer，则我们说任务 kkk 与任务 ttt 相似；否则，任务 kkk 与任务 ttt 不相似：

我们使用验证集来检查 Eq. 9 是否成立。具体而言，如果迁移模型 fk→tf_{k\rarr t}fk→t​ 对于任务 ttt 的验证集的分类效果比参照模型 f∅f_\varnothingf∅​ 的好，我们就说 kkk 包含有可共享的先验知识，相比于没有该知识的模型 f∅f_\varnothingf∅​，能够帮助 ttt 习得更好的模型，也就意味着积极的知识迁移（positive knowledge transfer）。我们置 TSV(t)[k]=1TSV^{(t)}[k]=1TSV(t)[k]=1 ，指示 kkk 与 ttt 相似；否则置 TSV(t)[k]=0TSV^{(t)}[k]=0TSV(t)[k]=0 ，指示 kkk 与 ttt 不相似。
