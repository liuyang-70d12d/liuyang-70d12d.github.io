Continual Local Training for Better Initialization of Federated Models

Xin Yao, Lifeng Sun

ICIP 2020

[paper] [code]

Main Idea

文章提出了一种在本地持续学习的联邦学习策略 FedCL。全局模型参数的重要性权重是由中心服务器上的一个 small proxy dataset 来评估决定的，并以此来限制各 client 本地的训练。这种方法的通信代价只增加了5%。

Related Work

喷了一下 FedCurv [11]，需要传输的参数数量是 FedAvg 的三倍。

Method

总的来说，FedCL 走的是用参数正则来实现持续学习的路子，然后把它应用到联邦学习当中。之前有看到，EWC[12]、MAS[16] 以及 SI[17] 都是这一类方法来做持续学习，FedCurv[11] 也是把 EWC 化用到了联邦学习当中。本篇文章相当于是对FedCurv的一个改进。

那这篇文章是怎么做的改进呢？先从 FedCurv 说起。FedCurv是在各个client上估计模型参数的重要性权重并相互交换这些信息，这样会带来至少两倍的额外通信开销。而 FedCL 是在中心服务器上使用 proxy dataset 估计重要性权重，然后将这些信息分发给各个 client。即便是在最糟糕的情况下，也只增加了50%的通信开销。文章在此基础上发现，重要性权重其实改变得并不频繁，所以它们可以按一定间隔来估算和分发，这种改进将进一步减小额外的通信开销至5%。FedCL兼容于各种参数正则的持续学习方法，如EWC[12]、MAS[16] 以及 SI[17] 等。文章中以EWC为例。

Estimate Importance Matrix on Proxy Data

FedCL 的估计模型参数的重要性权重的方法完全就是按EWC来搞的。具体而言，给定数据点 (xk,yk)∈Dproxy(x_k,y_k)\in\mathcal{D}_{proxy}(xk​,yk​)∈Dproxy​ （其中 Dproxy\mathcal{D}_{proxy}Dproxy​ 即为中心服务器上的 proxy dataset）以及全局模型参数 θg\theta_gθg​ ，那么重要性权重可以被近似为：

ωk=∣∣∂L(θg(xk),yk)∂θg∣∣2(1)\omega^k=||\cfrac{\partial\mathcal{L}(\theta_g(x_k),y_k)}{\partial\theta_g}||_2\qquad\qquad(1) ωk=∣∣∂θg​∂L(θg​(xk​),yk​)​∣∣2​(1)

通过累加在 proxy dataset上求得的 ωk\omega^kωk ，我们可以得到参数 θg\theta_gθg​ 对应的重要性权重 Ω\OmegaΩ：

Ω=1∣Dproxy∣∑(xk,yk)∈Dproxyωk(2)\Omega=\cfrac{1}{|\mathcal{D}_{proxy}|}\sum_{(x_k,y_k)\in\mathcal{D}_{proxy}}\omega^k\qquad\qquad(2) Ω=∣Dproxy​∣1​(xk​,yk​)∈Dproxy​∑​ωk(2)

Ω\OmegaΩ 就是我们所要得到的重要性权重矩阵。

Continual Training on Clients

接下来，作者对基于参数正则的持续学习方法和联邦学习做了一个溯源本质的探讨，说得极好，原文摘录如下：

The basic assumption of parameter-regularization continual learning algorithms is that deep neural networks are over-parameterized so that it is able to find an optimal solution θB∗\theta_B^*θB∗​ for task B that is close to the previously found solution for task A, θA∗\theta^*_AθA∗​. In FL, it is to find an optimal solution θ∗\theta^*θ∗ for both the local and global data distributions while keeping the generalization ability of the global model θg∗\theta_g^*θg∗​.

在一个包含 KKK 个client的联邦学习系统中，我们在每个client k∈C⋅Kk\in C\cdot Kk∈C⋅K 上优化下面的经验风险函数（empirical risk）：

Lk(θk)=Llocal(θk)+λ∑i,jΩij(θk,ij−θg,ij)2(3)\mathcal{L}_k(\theta_k)=\mathcal{L}_{local}(\theta_k)+\lambda\sum_{i,j}\Omega_{ij}(\theta_{k,ij}-\theta_{g,ij})^2\qquad\qquad(3) Lk​(θk​)=Llocal​(θk​)+λi,j∑​Ωij​(θk,ij​−θg,ij​)2(3)

(3) 式具体什么意思就不细说了，毕竟和EWC以及FedCurv太像了。

Cut Down Bandwidth

按照前面提到的方法设计，（相比于FedAvg）FL系统每个round需要额外增加50%的通信开销。（从client向server上传的时候传输量没变，而从server向client传输的时候还要带上和模型参数同样大小的Ω\OmegaΩ矩阵 。）

但其实全局模型参数变化得并不剧烈，可以考虑每隔 NNN 个round再重新估计和分发 Ω\OmegaΩ 矩阵。假设 N=10N=10N=10，那么通信开销平均到每个round就只增加了5% 。
