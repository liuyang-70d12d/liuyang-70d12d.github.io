Federated Learning For Keyword Spotting

ICASSP 2019

David Leroy, Alice Coucke, Thibaut Lavril, Thibault Gisselbrecht, Joseph Dureau

Paper Adress

Background

首先介绍一下“唤醒词检测”（wake word detection）。

Wake word detection is used to start an interaction with a voice assistant. A specific case of keyword spotting (KWS), it continuously listens to an audio stream to detect a predefined keyword or set of keywords. Well-known examples of wake words include Apple’s “Hey Siri” or Google’s “OK Google”. Once the wake word is detected, voice input is activated and processed by a spoken language understanding engine, powering the perception abilities of the voice assistant.

Wake word detectors usually run on device in an alwayson fashion, which brings two major difficulties.

First, it should run with minimal memory footprint and computational cost.

Secondly, the wake word detector should behave consistently in any usage setting, and show robustness to background noise. Robustness also implies a strong speaker variability coverage. Speaker variability remains a major challenge. High accuracy is all the more important since the model can be triggered at any time: it is therefore expected to capture most of the commands (high recall) while not triggering unintentionally (low false alarm rate).

一句话概括：这篇文章是在用联邦学习来做“唤醒词检测”。

Main Idea

In this work, we investigate the use of federated learning (FL) in the context of an embedded wake word detector.

Experiments

Dataset

The reference dataset for multi-class keyword spotting is the speech command dataset [3], but the speech command task is generally preceded by a wake word detector and is focused on minimizing the confusion across classes, not robustness to false alarms.

We constituted a crowdsourced dataset[4] for the Hey Snips wake word.

== == == == == == == == 下面是对这个数据集的介绍 == == == == == == ==

The data used here was collected from 1.8k contributors that recorded themselves on their device with their own microphone while saying several occurrences of the Hey Snips wake word along with negative short sentences from various text sources (e.g. subtitles).

This crowdsourcing-induced data distribution mimicks a real-world non-i.i.d, unbalanced and highly distributed setting, and a parallel is therefore drawn in the following work between a crowdsourcing contributor and a voice assistant user. The statistics about the dataset comforting this analogy are summarized in Table 1. The train, dev and test splits are built purposely using distinct users, 77% of users being used solely for training while the remaining are used for parameter tuning and final evaluation, measuring the generalization power of the model to new users.

Table 1. Dataset statistics for the Hey Snips wake word - 18% of utterances are positive, with strong per user imbalance in the number of utterances (mean: 39, standard dev: 32)

Model

Acoustic features are generated based on 40-dimensional mel-frequency cepstrum coefficients (MFCC) computed every 10ms over a window of 25ms. The input window consists in 32 stacked frames, symmetrically distributed in left and right contexts.

The architecture is a CNN with 5 stacked dilated convolutional layers of increasing dilation rate, followed by two fully-connected layers and a softmax inspired from [5]. The total number of parameters is 190,852, initialized using Xavier initialization. The model is trained using cross-entropy loss on frames prediction.

Conclusion

In this work, we investigate the use of federated learning on crowdsourced speech data to learn a resource-constrained wake word detector. We show that a revisited Federated Averaging algorithm with per-coordinate averaging based on Adam in place of standard global averaging allows the training to reach a target stopping criterion of 95% recall per 5 FAH within 100 communication rounds on our crowdsourced dataset for an associated upstream communication costs per client of 8MB.
