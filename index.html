Paper Note: Continual Local Training For Better Initialization of Federated Models

Created2020-12-10Updated2020-12-11

Continual Local Training for Better Initialization of Federated Models

Xin Yao, Lifeng Sun

ICIP 2020

[paper] [code]

Main Idea

文章提出了一种在本地持续学习的联邦学习策略 FedCL。全局模型参数的重要性权重是由中心服务器上的一个 small proxy dataset 来评估决定的，并以此来限制各 client 本地的训练。这种方法的通信代价只增加了5%。

Related Work

喷了一下 FedCurv [11]，需要传输的参数数量是 FedAvg 的三倍。

Method

总的来说，FedCL 走的是用参数正则来实现持续学习的路子，然后把它应用到联邦学习当中。之前有看到，EWC[12]、MAS[16] 以及 SI[17] 都是这一类方法来做持续学习，FedCurv[11] 也是把 EWC 化用到了联邦学习当中。本篇文章相当于是对FedCurv的一个改进。

那这篇文章是怎么做的改进呢？先从 FedCurv 说起。FedCurv是在各个client上估计模型参数的重要性权重并相互交换这些信息，这样会带来至少两倍的额外通信开销。而 FedCL 是在中心服务器上使用 proxy dataset 估计重要性权重，然后将这些信息分发给各个 client。即便是在最糟糕的情况下，也只增加了50%的通信开销。文章在此基础上发现，重要性权重其实改变得并不频繁，所以它们可以按一定间隔来估算和分发，这种改进将进一步减小额外的通信开销至5%。FedCL兼容于各种参数正则的持续学习方法，如EWC[12]、MAS[16] 以及 SI[17] 等。文章中以EWC为例。

Ω=1∣Dproxy∣∑(xk,yk)∈Dproxyωk(2)\Omega=\cfrac{1}{|\mathcal{D}_{proxy}|}\sum_{(x_k,y_k)\in\mathcal{D}_{proxy}}\omega^k\qquad\qquad(2) Ω=∣Dproxy​∣1​(xk​,yk​)∈Dproxy​∑​ωk(2)

Paper Note: Memory Aware Synapses: Learning what (not) to forget

Created2020-12-09Updated2020-12-10

Memory Aware Synapses: Learning what (not) to forget

Rahaf Aljundi, Francesca Babiloni, Mohamed Elhoseiny, Marcus Rohrbach, Tinne Tuytelaars

ECCV 2018

[paper]

Background

文章有个很有意思的提法：在实际情况下，持续学习所要学习的任务是希望是无限多的，但是模型的容量却是有限的，或早或晚总要消耗完。所以持续学习是需要做一个折衷妥协的，即不能指望模型“记住”所有的知识，需要指出模型应该“忘记”哪些知识。就像那句歌词，“ 挥别错的才能和对的相逢”，表达了类似的意思吧：）。比如说，某处的摄像头只需要识别它所处场景下的物体即可，其它场景下的“知识”就大可以忘掉了，这样该摄像头的模型才能有更多的容量或者说能力来学习当下场景的知识。也就是说，我们希望持续学习应该有根据未标记的测试数据（unlabeled test data）来选择（不）忘记一些知识的能力——经常出现的数据，“记忆”的印象更深刻一些；不经常出现的数据，就慢慢忘掉就好了。

Main Idea

文章提出了一种新的持续学习方法，称为 Memory Aware Synapses (MAS)。MAS相比于之前方法的亮眼之处在于它能够通过未标记的数据（unlabeled data）来学习到模型的哪一部分是重要的——也就是通过无监督、在线的方式评估参数的重要性。模型因此可以适应某些特定的测试场景，并且能够连续地更新重要权重。这种模型不是通过损失函数来估计模型参数的重要性，而是通过输出函数的灵敏度（the sensitivity of the output function）得到的。

文章指出，现存（文章最后修改于2018年10月）的持续学习方法都不太行，都没有满足人们对持续学习所有期望的特征（见 Table 1）。那么人们期望的持续学习特征有哪些呢？文章列了下面这几条：(1) 内存消耗应该是一个常量，不能随着所学习的任务数量增多，内存消耗也跟着涨；(2) 与任务无关，不能只能应用于分类，其它任务就用不了了；(3) 已经预训练的模型可以拿来在它上面继续训练新的任务；(4) 能够从无标签数据中学习，使之能够适应之前训练时没有遇到过的数据（场景）；(5) 能够适应性地学习到该“忘记”什么，该“记住”什么，以便给以后的任务留有空间。

文章将相关的工作分为两个大类：基于数据的（data-based）和基于模型的（model-based）。**基于数据的方法（Data-based approaches）**是用新任务的数据来估计之前任务的性能表现。当各个任务之间的数据分布差异不太大时，这种方法的效果会比较好。这种方法一般是用于持续学习分类任务。基于模型的方法（Model-based approaches） 则聚焦于模型的参数，不care任务的具体数据。这篇文章即属于这种方法，属于此方法的工作还有EWC [12]、SI[39]。本文与[12]、[39]非常相似，都是给每个网络参数计算一个重要性权重，作为正则项加在损失函数里，然后对重要参数的改变进行惩罚。区别仅在于计算重要性权重的方法不同。

文章随之喷了SI[39]的三个缺点，其中一个是说SI不适用于已经预训练的模型。因为已经预训练的模型许多参数可能不会再有大的变化，使得这些参数的重要性被低估。

Method

前面已经说过，本文与[12]、[39]非常相似，那就重点说说本文的重要性权重是如何得到的。

Estimating parameter importance

我们可以想到，一个模型其实是在对一个很ideal的函数 Fˉ\bar{F}Fˉ（论文里叫做 true function）做近似，即 FFF 。近似函数 FFF 将输入 X1X_1X1​ 映射到输出 Y1Y_1Y1​ 上。持续学习的目的就是习得尽可能精确的 FFF，并在后续训练其它任务时依然保留有 FFF。现在我们来看看，函数 FFF 的输出对于网络参数的改变有多敏感。一个在参数 θ={θij}\theta=\{\theta_{ij}\}θ={θij​} 上的小扰动 δ={δij}\delta=\{\delta_{ij}\}δ={δij​} 对于函数 FFF 输出的改变可以用下式近似表示：

F(xk;θ+δ)−F(xk;θ)≈∑i,jgij(xk)δij(1)F(x_k;\theta+\delta)-F(x_k;\theta)\approx\sum_{i,j}g_{ij}(x_k)\delta_{ij}\qquad\qquad(1) F(xk​;θ+δ)−F(xk​;θ)≈i,j∑​gij​(xk​)δij​(1)

其中 gij(xk)=∂(F(xk;θ))∂θi,jg_{ij}(x_k)=\cfrac{\partial(F(x_k;\theta))}{\partial\theta_{i,j}}gij​(xk​)=∂θi,j​∂(F(xk​;θ))​ 是在数据点 xkx_kxk​ 上估计得到的习得函数 FFF 关于参数 θij\theta_{ij}θij​ 的梯度。

进而，我们可以使用梯度 gijg_{ij}gij​ 的大小来表征一个参数的重要性，也就是对于数据点 xkx_kxk​ 而言，在这个参数上施加一个小扰动会在多大程度上改变习得函数的输出。我们将对应于所有数据点的梯度加总起来，从而得到 θij\theta_{ij}θij​ 的重要性权重 Ωij\Omega_{ij}Ωij​ ：

Ωij=1N∑k=1N∣∣gij(xk)∣∣(2)\Omega_{ij}=\cfrac{1}{N}\sum_{k=1}^N||g_{ij}(x_k)||\qquad\qquad(2) Ωij​=N1​k=1∑N​∣∣gij​(xk​)∣∣(2)

Learning a new task

当我们要训练新任务 TnT_nTn​ 时，对应有了新任务损失函数 Ln(θ)L_n(\theta)Ln​(θ) ，整个损失函数为：

Continual Learning Through Synaptic Intelligence

Friedemann Zenke, Ben Poole, Surya Ganguli

ICML 2017

[paper] [video] [code]

Related Work

这篇文章将从2013年~2017年解决“灾难性遗忘”（catastrophic forgetting）的方法分为了三类 (1) architectural， (2) functional，以及 (3) structural approaches。

其中 functional approaches 包括 Li & Hoiem (2016)(LwF) 以及 Jung et al. (2016)。这一类方法的问题在于计算代价比较大，新的任务的数据都需要在旧任务对应的网络跑一遍前向传播。

文章还谈到了EWC的问题。EWC需要计算 the diagonal of the Fisher，需要把所有可能的输出 label 都加起来。也就是说，计算 the diagonal of the Fisher 的复杂度与输出的数量呈线性关系，从而限制了EWC方法只能应用于低维输出空间。

Method

我认为本文提出的方法相比于EWC最大的优点在于它是online的。

神经网络的训练过程可以用参数空间中的一段轨迹 θ(t)\boldsymbol{\theta}(t)θ(t) 来刻画。成功的持续学习的训练过程应该使这段轨迹的端点都落在每个所训练任务的loss函数的某个最小值附近。

我们先来考虑时间 ttt 时刻，一个极小量的参数更新 δ(t)\boldsymbol{\delta}(t)δ(t) 。相对应的，损失函数的改变值可以用梯度 g=∂L∂θ\boldsymbol{g}=\cfrac{\partial L}{\partial\boldsymbol{\theta}}g=∂θ∂L​ 来做近似：

Some Thoughts

整篇文章的核心在 (4) 和 (5) 两个式子，文章说到底就是提出一个新的损失函数，这个损失函数可以根据文章所认为的参数重要性准则来对网络模型的参数做正则化。从(4) 和 (5) 式想到，其实对于联邦学习而言，参数聚合不一定就是client上传模型参数，server加权平均然后再下发参数给client那么粗暴，为了保证聚合参数对client的提升效果，可以在client本地借鉴一下本文的想法，在本地原有模型的基础上向从server端得到的模型进行渐进式的更新。

Background

人类和动物是可以持续不断地学习新知识，而不会忘了旧知识的。根据相关研究表明，哺乳动物通过将知识保存在新脑皮层（neocortical circuits）中以避免遗忘。当老鼠掌握一项新技能时，一部分兴奋性突触就会增强，这表现为神经元的某些树突的体积增加。重要的是，尽管随后学习了其他任务，这些扩大的树突仍然存在，这说明几个月后老鼠仍然掌握着这些技能。当这些树突被有选择地“擦除”时，相应的技能就会被遗忘。

这些实验发现表明，新脑皮层的持续学习依赖于任务相关的突触巩固。通过使一定比例的突触可塑性降低能够在很长时间范围内稳定持久地编码知识。

Main Idea

论文提出了一种新的持续学习算法 Elastic Weight Consolidation (EWC)，该方法可以看作是在神经网络中模拟了突触巩固。简言之，该方法是把在之前任务中重要的权重的学习率降低——对于之前任务重要的参数变化得慢一点，不重要的参数变化得快一点。

Method

深度神经网络是过度参数化的（overparameterization），也就是说不同参数的神经网络可能有着相同的性能表现。这一理论前提提供了一种可能——适应于任务A的网络 θA∗\theta_A^*θA∗​ 与适应于任务B的网络 θB∗\theta_B^*θB∗​ 可能非常接近。换句话说，如果在持续学习的场景下，如果我们已经找到适应于任务A的网络 θA∗\theta_A^*θA∗​ ，那么我们就很有可能在不太多改变 θA∗\theta_A^*θA∗​ 的情况下找到适应于任务B的网络 θB∗\theta_B^*θB∗​ ，从而使得 θB∗\theta_B^*θB∗​ 在任务A上的表现依然比较好。这种限制以二次惩罚项的形式出现，这样做的好处在于仅仅对范数的限制，可以保证在以 θA∗\theta_A^*θA∗​ 为中心锚点的一定范围内改变某些参数——对之前任务影响大的参数改变少一点，反之改变多一点，所谓的 “elastic” 就体现在这里。

Evaluation

论文对EWC做了一些实验，其中一个是探究EWC是将对不同任务重要的参数都分离开了还是说其实这些任务都是共用了一些相同的参数来做表征。论文通过实验得到两点结论：(1) 各任务间的数据越相似，它们共用的参数越多，Fisher information matrices 的重叠部分（overlap）就越大；(2) 对于任务数据相似性不大的网络而言，随着网络层数加深，共用的参数越多，Fisher information matrices 的重叠部分（overlap）越大。

Background

和 Learning without Forgetting (LwF) 那篇一样，先拿以下三种老方法祭旗开路：

**Feature extraction：**在遇到新任务时，模型参数不改变，相当于只是对新任务的数据做表征，做特征提取。对旧任务友好，对新任务不友好。

**Fine-tuning：**遇到新任务时，模型在旧任务的基础上做fine-tune，容易学了后面忘了前面。对旧任务不友好，对新任务友好。 

**Joint training：**把所有任务的数据汇到一起，一块训练。这显然是不现实的，不然也不需要持续学习了。提到 Joint training 的意义也仅在于把它作为持续学习的理论上界而已。

对于本文而言，比较关键的两篇工作：LwF [14] 和 EWC [12]

Learning without Forgetting (LwF): proposes a way to train convolutional networks, where a shared model is used for the different tasks but with separate classification layers. When a new task is presented, a new classification layer is added. Then, the model is fine-tuned on the data of the new task, with an additional loss that incorporates the knowledge of the old tasks. This loss tries to keep the previous task predictions on the new data unchanged. Such a solution reduces the forgetting but is heavily relying on the new task data. As a consequence, it suffers from a build-up of errors when facing a sequence of tasks [1].

LwF proposes to preserve the previous performance through the knowledge distillation loss introduced in [11]. They consider a shared convolutional network between the different tasks in which only the last classification layer is task specific. When encountering a new task, the outputs of the existing classification layers given the new task data are recorded. During training, these outputs are preserved through a modified cross-entropy loss that softens the class probabilities in order to give a higher weight to the small outputs. This method reduces the forgetting, especially
when the datasets come from related manifolds. Nevertheless, it has been shown by [22] that LwF suffers from a build up of errors in a sequential scenario where the data comes from the same environment. Similarly, [1] shows that the LwF performance drops when the model is exposed to a sequence of tasks drawn from different distributions.

Elastic Weight Consolidation (EWC): Rather than having a data-oriented analysis, they consider the knowledge gained in the model itself, and transfer it from one task to another in a Bayesian update fashion. The method relies on approximating the weight distribution after training the model on the new task. A Gaussian distribution, for which the mean is given by the optimal weights for the first task, and the variance given by the diagonal of the Fisher information matrix is used as an approximation. Such a solution is based on a strong principle and gives interesting results. However, it requires to store a number of parameters that is comparable to the size of the model itself.

EWC is to constrain the weights WiW_iWi​ while training for a second task with an additional loss ∑iλ2Fi(Wi−W1,i∗)\sum_i\cfrac{\lambda}{2}F_i(W_i-W_{1,i}^*)∑i​2λ​Fi​(Wi​−W1,i∗​) where W1,i∗W_{1,i}^*W1,i∗​ are the optimal weights of the first task, and FiF_iFi​ the diagonal terms of their Fisher information matrix. The use of the Fisher matrix prevents the weights that are important for the first task to change much. In our point of view, this method, despite its success, has two drawbacks. First, the method keeps the weights in a neighborhood of one possible minimizer of the empirical risk of the first task. However, there could be another solution that can give a better compromise between the two tasks.（这个问题提得漂亮！） Second, it needs to store a large number of parameters that grows with the total number of weights and the number of tasks. For these reasons, rather than constraining the weights, we choose to constrain the resulting features, enforcing that those that are important for the previous tasks do not change much. By constraining only a sub-manifold of the features, we allow the weights to adjust so as to optimize the features for the new task, while preserving those that ensure a good performance on the previous tasks.

Main Idea

We introduce the use of autoencoders as a tool to preserve the knowledge from one task while learning another. For each task, an undercomplete autoencoder is trained after training the task model. It captures the most important features for the task objective. When facing a new task, this autoencoder is used to ensure the preservation of those important features. This is achieved by defining a loss on the reconstructions made by the autoencoder. In this manner, we only restrict a subset of the features to be unchanged while we give the model the freedom to adapt itself to the new task using the remaining capacity. Background

We aim at developing a simple but effective strategy on a variety of image classification problems with Convolutional Neural Network (CNN) classifiers. In our setting, a CNN has a set of shared parameters θs\theta_sθs​ (e.g., five convolutional layers and two fully connected layers for AlexNet [3] architecture), task-specific parameters for previously learned tasks θo\theta_oθo​ (e.g., the output layer for ImageNet [4] classification and corresponding weights), and randomly initialized task-specific parameters for new tasks θn\theta_nθn​ (e.g., scene classifiers). It is useful to think of θo\theta_oθo​ and θn\theta_nθn​ as classifiers that operate on features parameterized by θs\theta_sθs​. Currently, there are three common approaches (Figures 1, 2) to learning θn\theta_nθn​ while benefiting from previously learned θs\theta_sθs​:

  • Feature Extraction: θs\theta_sθs​ and θo\theta_oθo​ are unchanged, and the outputs of one or more layers are used as features for the new task in training θn\theta_nθn​.

We address the problem of adapting a vision system to a new task while preserving performance on original tasks, without access to training data for the original tasks. We here demonstrate that replay can be a powerful tool in continual learning. We propose a simple technique, Continual Learning with Experience And Replay (CLEAR), mixing on-policy learning from novel experiences (for plasticity) and off-policy learning from replay experiences (for stability).

CLEAR 比较亮眼的一个特点：

No task information. CLEAR does not rely upon the identity of tasks or the boundaries between them, by contrast with prior art. This means that CLEAR can be applied to a much broader range of situations, where tasks are unknown or are not discrete.

In settings where data is streaming without clear task boundaries, Rolnick et al. [45] suggest reservoir sampling to limit the number of stored samples to a fixed budget assuming an overall i.i.d. distributed data stream.
