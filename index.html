Paper Note: Overcoming Catastrophic Forgetting by Incremental Moment Matching

Created2020-12-16Updated2020-12-17

Overcoming Catastrophic Forgetting by Incremental Moment Matching

Sang-Woo Lee, Jin-Hwa Kim, Jaehyun Jun, Jung-Woo Ha, Byoung-Tak Zhang

NIPS 2017

[paper] [code] [video]

Main Idea

文章提出了一种解决灾难性遗忘的方法，incremental moment matching (IMM) 。IMM使用了 Bayesian neural networks 的分析框架，这意味着在神经网络的参数中引入了不确定性，并且计算了后验分布。后验分布中随机变量的维数是神经网络中参数的数量。

IMM将混合高斯的后验做了近似，每个分量代表单个任务的参数，混合高斯的后验近似于一系列任务组合的一个高斯分布。 为了合并后验概率，我们介绍两种矩匹配方法。 一种是mean-IMM，它简单地对旧任务和新任务的两个网络的参数取平均值，以使任务组合的一个近似后验分布与每个单个任务的高斯后验之间的KL散度的平均值最小化。 另一个是mode-IMM，它通过使用拉普拉斯逼近法（Laplacian approximation）近似表示两个高斯后验的混合模式（表征了两个网络的参数）来合并两个网络的参数。

通常，将整个任务系列对应的最终后验分布假设为高斯分布过于朴素了。为了使我们的IMM有效，后验均值（即 Figure 1 中的 μ1\mu_1μ1​ 和 μ2\mu_2μ2​）之间的损失函数的搜索空间需要是光滑的和凸状的（convex-like）。 换句话说，在新旧任务的两个网络的均值之间不应存在高成本障碍。 为了使我们对神经网络的高斯分布的假设合理，也就是为了找到 μ2\mu_2μ2​ 与 μ1\mu_1μ1​ 之间光滑、凸状的通路，我们在使用IMM的过程中应用了三种迁移学习技术：weight transfer，新旧参数之间的L2-norm，以及我们新提出的使用旧参数的dropout的变种。 IMM的整个过程如Figure 1所示。

Figure 1. mean-IMM 仅仅是平均了两个神经网络的参数，而 mode-IMM 是在试图寻找混合高斯后验的最大值。

Paper Note: PackNet: Adding Multiple Tasks to a Single Network by Iterative Pruning

PackNet: Adding Multiple Tasks to a Single Network by Iterative Pruning

Arun Mallya, Svetlana Lazebnik

CVPR 2018

[paper]

Main Idea

文章提出了一种新的持续学习方法，即以对准确率尽可能小的损失为代价，使用基于权重的剪枝技术来释放掉训练完一个任务后冗余的模型参数。然后将未被减枝的参数固定住，被减枝的参数就会被用来学习新的任务。通过使用由剪枝得到的 task-specific parameter masks，我们的模型即便在经历了多个任务之后依然能够保持相同水准的准确率，并且对于每个新任务而言只产生很小的存储开销。

Figure 1. 以一个 5×5 的 filter 为例说明本方法的训练过程。(a) 在任务 I 上进行初始训练，学习得到一个完整的filter；(b) 裁剪掉 60% (15/25) 再重新训练，得到了一个对应于任务 I 的稀疏的 filter，其中白圆圈代表0值权重，对应于任务 I 的权重将被固定，并且不会在后续过程中被裁剪；© 被剪枝的权重可以用于学习任务 II；(d) 再减枝33%(5/15)并且重新训练，橘色的圆圈即代表对应于任务 II 的权重，随即也被固定，并且之后不会被剪枝。这个过程将会随着学习完所有任务或者使用完所有filter 而终止。在推断时，根据所选的任务使用对应的mask。

Paper Note: Federated Continual Learning with Weighted Inter-client Transfer

Created2020-12-11Updated2020-12-13

Federated Continual Learning with Weighted Inter-client Transfer

Jaehong Yoon, Wonyong Jeong, Giwoong Lee, Eunho Yang, Sung Ju Hwang

Under review as a conference paper at ICLR 2021

[paper (arxiv)] [paper (ICLR 2021)]

Background

之前的持续学习都是模型只能学习到自己“亲身经历”过的任务，而人是可以通过间接的方式去学习的，比如通过与其他人的口头交流、书籍或其它媒体等。文章因此引出一个疑问：那么不同client上的各模型能不能直接去学习其它client上的模型已经训练过的任务所得到的知识？也就是说，client A训练过 task T，但是client B没有，那client B能否不直接训练 task T，而直接从 client A习得。这个想法的问题也是显然的，即数据隐私性问题和通信代价问题等，也由此自然而然地引入了联邦学习。该问题在文中被称为 Federated Continual Learning (FCL)。

FCL 的问题也是显而易见的——联邦学习和持续学习面临的问题也就是它都需要面对的问题：不仅要面对灾难性遗忘问题，还要面对某个client要被其它client干扰的问题（按我的理解就是说参数聚合的问题，聚合完以后成了“四不像”。不过后文有它自己的解释）。

Figure 1.(b)

如 Figure 1.(b) 所示，对于某个训练MNIST数据集的client，如果从一个训练相关任务（SVHN数据集）的client处获取到知识（如图中绿色箭头所示），那么它的起始准确率就很高，收敛速度也更快，并最终能获得更高的准确率；而如果从一个与目标任务很不相同（CIFAR-10）的client处获取到知识（如图中红色箭头所示），那么该client的表现可能比不获取知识时的模型还要差。

既然如此，我们的目标就很明确了，就是要有选择性地使用其它client的“知识”，把client间的干扰（inter-client interference）搞得少少的，把client间的知识传递（inter-client knowledge transfer）搞得多多的。

联邦学习的另外一个问题就是通信开销的问题，尤其对于本文而言，如果要使用其它client的知识，通信开销会尤其得大。这也就要求我们对“知识”的表示越紧凑越好。

Main Idea

文章提出了一种新的联邦持续学习框架 Federated Weighted Inter-client Transfer (FedWeIT)，它将client的本地模型参数分解为了两部分——稠密的基本参数（base parameter）以及稀疏的适应于任务（task-adaptive）的参数。FedWeIT 是如何解决任务间的“干扰”问题的呢？基本参数编码了任务通用（task-generic）的知识，适应于任务的参数编码了任务相关（task-specific）的知识。当我们在使用通用知识的时候，也希望模型能够有选择性地使用来自其它client的任务相关的知识。为此，我们可以使本地模型对中心服务器广播的适应于任务的参数进行加权组合，由此选择对当前任务有帮助的任务相关的知识。

FedWeIT 的通信效率也很高，因为适应于任务的参数是高度稀疏的而且只需要在被创建的时候传输一次。

Method

首先，联邦持续学习不适合于原有的联邦学习框架，如 FedAvg (McMahan et al., 2016) 和 FedProx (Li et al., 2018)，因为这些联邦学习框架解决不了持续学习的灾难性遗忘问题。如果想要能够联邦持续学习，首先要解决的就是如何在联邦学习场景下解决灾难性遗忘问题。

不仅如此，对于联邦学习场景而言，还有两个更具体的问题：inter-client interference 和 communication-efficiency。inter-client interference 问题是说 client cic_ici​ 在第 qqq 轮的任务 Ti(q)\mathcal{T}_i^{(q)}Ti(q)​ 可能与 client cjc_jcj​ 在第 rrr 轮的任务 Tjr\mathcal{T}_j^{r}Tjr​ 的任务非常相似，如果用过去的参数聚合方法就显得过于粗暴，也就是如果没有对应好这些任务的相似性，可能就失去了“联邦”的意义，甚至会起到反向的阻碍作用。另外一个就是 communication-efficiency 问题，持续学习所要学习的任务量会不断增加，对应的各client本地的模型大小、传输成本会不会随之增加，这也是需要考虑的一个问题。

[paper] [code]

Main Idea

文章提出了一种在本地持续学习的联邦学习策略 FedCL。全局模型参数的重要性权重是由中心服务器上的一个 small proxy dataset 来评估决定的，并以此来限制各 client 本地的训练。这种方法的通信代价只增加了5%。

Related Work

喷了一下 FedCurv [11]，需要传输的参数数量是 FedAvg 的三倍。

Method

总的来说，FedCL 走的是用参数正则来实现持续学习的路子，然后把它应用到联邦学习当中。之前有看到，EWC[12]、MAS[16] 以及 SI[17] 都是这一类方法来做持续学习，FedCurv[11] 也是把 EWC 化用到了联邦学习当中。本篇文章相当于是对FedCurv的一个改进。

那这篇文章是怎么做的改进呢？先从 FedCurv 说起。FedCurv是在各个client上估计模型参数的重要性权重并相互交换这些信息，这样会带来至少两倍的额外通信开销。而 FedCL 是在中心服务器上使用 proxy dataset 估计重要性权重，然后将这些信息分发给各个 client。即便是在最糟糕的情况下，也只增加了50%的通信开销。文章在此基础上发现，重要性权重其实改变得并不频繁，所以它们可以按一定间隔来估算和分发，这种改进将进一步减小额外的通信开销至5%。FedCL兼容于各种参数正则的持续学习方法，如EWC[12]、MAS[16] 以及 SI[17] 等。文章中以EWC为例。

Ω=1∣Dproxy∣∑(xk,yk)∈Dproxyωk(2)\Omega=\cfrac{1}{|\mathcal{D}_{proxy}|}\sum_{(x_k,y_k)\in\mathcal{D}_{proxy}}\omega^k\qquad\qquad(2) Ω=∣Dproxy​∣1​(xk​,yk​)∈Dproxy​∑​ωk(2)

Rahaf Aljundi, Francesca Babiloni, Mohamed Elhoseiny, Marcus Rohrbach, Tinne Tuytelaars

[paper]

Background

文章有个很有意思的提法：在实际情况下，持续学习所要学习的任务是希望是无限多的，但是模型的容量却是有限的，或早或晚总要消耗完。所以持续学习是需要做一个折衷妥协的，即不能指望模型“记住”所有的知识，需要指出模型应该“忘记”哪些知识。就像那句歌词，“ 挥别错的才能和对的相逢”，表达了类似的意思吧：）。比如说，某处的摄像头只需要识别它所处场景下的物体即可，其它场景下的“知识”就大可以忘掉了，这样该摄像头的模型才能有更多的容量或者说能力来学习当下场景的知识。也就是说，我们希望持续学习应该有根据未标记的测试数据（unlabeled test data）来选择（不）忘记一些知识的能力——经常出现的数据，“记忆”的印象更深刻一些；不经常出现的数据，就慢慢忘掉就好了。

Main Idea

文章提出了一种新的持续学习方法，称为 Memory Aware Synapses (MAS)。MAS相比于之前方法的亮眼之处在于它能够通过未标记的数据（unlabeled data）来学习到模型的哪一部分是重要的——也就是通过无监督、在线的方式评估参数的重要性。模型因此可以适应某些特定的测试场景，并且能够连续地更新重要权重。这种模型不是通过损失函数来估计模型参数的重要性，而是通过输出函数的灵敏度（the sensitivity of the output function）得到的。

Related Work

Table 1

文章指出，现存（文章最后修改于2018年10月）的持续学习方法都不太行，都没有满足人们对持续学习所有期望的特征（见 Table 1）。那么人们期望的持续学习特征有哪些呢？文章列了下面这几条：(1) 内存消耗应该是一个常量，不能随着所学习的任务数量增多，内存消耗也跟着涨；(2) 与任务无关，不能只能应用于分类，其它任务就用不了了；(3) 已经预训练的模型可以拿来在它上面继续训练新的任务；(4) 能够从无标签数据中学习，使之能够适应之前训练时没有遇到过的数据（场景）；(5) 能够适应性地学习到该“忘记”什么，该“记住”什么，以便给以后的任务留有空间。

文章将相关的工作分为两个大类：基于数据的（data-based）和基于模型的（model-based）。**基于数据的方法（Data-based approaches）**是用新任务的数据来估计之前任务的性能表现。当各个任务之间的数据分布差异不太大时，这种方法的效果会比较好。这种方法一般是用于持续学习分类任务。基于模型的方法（Model-based approaches） 则聚焦于模型的参数，不care任务的具体数据。这篇文章即属于这种方法，属于此方法的工作还有EWC [12]、SI[39]。本文与[12]、[39]非常相似，都是给每个网络参数计算一个重要性权重，作为正则项加在损失函数里，然后对重要参数的改变进行惩罚。区别仅在于计算重要性权重的方法不同。

文章随之喷了SI[39]的三个缺点，其中一个是说SI不适用于已经预训练的模型。因为已经预训练的模型许多参数可能不会再有大的变化，使得这些参数的重要性被低估。

Estimating parameter importance

我们可以想到，一个模型其实是在对一个很ideal的函数 Fˉ\bar{F}Fˉ（论文里叫做 true function）做近似，即 FFF 。近似函数 FFF 将输入 X1X_1X1​ 映射到输出 Y1Y_1Y1​ 上。持续学习的目的就是习得尽可能精确的 FFF，并在后续训练其它任务时依然保留有 FFF。现在我们来看看，函数 FFF 的输出对于网络参数的改变有多敏感。一个在参数 θ={θij}\theta=\{\theta_{ij}\}θ={θij​} 上的小扰动 δ={δij}\delta=\{\delta_{ij}\}δ={δij​} 对于函数 FFF 输出的改变可以用下式近似表示：

F(xk;θ+δ)−F(xk;θ)≈∑i,jgij(xk)δij(1)F(x_k;\theta+\delta)-F(x_k;\theta)\approx\sum_{i,j}g_{ij}(x_k)\delta_{ij}\qquad\qquad(1) F(xk​;θ+δ)−F(xk​;θ)≈i,j∑​gij​(xk​)δij​(1)

其中 gij(xk)=∂(F(xk;θ))∂θi,jg_{ij}(x_k)=\cfrac{\partial(F(x_k;\theta))}{\partial\theta_{i,j}}gij​(xk​)=∂θi,j​∂(F(xk​;θ))​ 是在数据点 xkx_kxk​ 上估计得到的习得函数 FFF 关于参数 θij\theta_{ij}θij​ 的梯度。

进而，我们可以使用梯度 gijg_{ij}gij​ 的大小来表征一个参数的重要性，也就是对于数据点 xkx_kxk​ 而言，在这个参数上施加一个小扰动会在多大程度上改变习得函数的输出。我们将对应于所有数据点的梯度加总起来，从而得到 θij\theta_{ij}θij​ 的重要性权重 Ωij\Omega_{ij}Ωij​ ：

Ωij=1N∑k=1N∣∣gij(xk)∣∣(2)\Omega_{ij}=\cfrac{1}{N}\sum_{k=1}^N||g_{ij}(x_k)||\qquad\qquad(2) Ωij​=N1​k=1∑N​∣∣gij​(xk​)∣∣(2)

Related Work

这篇文章将从2013年~2017年解决“灾难性遗忘”（catastrophic forgetting）的方法分为了三类 (1) architectural， (2) functional，以及 (3) structural approaches。

其中 functional approaches 包括 Li & Hoiem (2016)(LwF) 以及 Jung et al. (2016)。这一类方法的问题在于计算代价比较大，新的任务的数据都需要在旧任务对应的网络跑一遍前向传播。

文章还谈到了EWC的问题。EWC需要计算 the diagonal of the Fisher，需要把所有可能的输出 label 都加起来。也就是说，计算 the diagonal of the Fisher 的复杂度与输出的数量呈线性关系，从而限制了EWC方法只能应用于低维输出空间。

Method

我认为本文提出的方法相比于EWC最大的优点在于它是online的。

神经网络的训练过程可以用参数空间中的一段轨迹 θ(t)\boldsymbol{\theta}(t)θ(t) 来刻画。成功的持续学习的训练过程应该使这段轨迹的端点都落在每个所训练任务的loss函数的某个最小值附近。

我们先来考虑时间 ttt 时刻，一个极小量的参数更新 δ(t)\boldsymbol{\delta}(t)δ(t) 。相对应的，损失函数的改变值可以用梯度 g=∂L∂θ\boldsymbol{g}=\cfrac{\partial L}{\partial\boldsymbol{\theta}}g=∂θ∂L​ 来做近似：

L(θ(t)+δ(t))−L(θ(t))≈∑kgk(t)δk(t)(1)L(\boldsymbol{\theta}(t)+\boldsymbol{\delta}(t))-L(\boldsymbol{\theta}(t))\approx\sum_k g_k(t)\delta_k(t)\qquad\qquad(1) L(θ(t)+δ(t))−L(θ(t))≈k∑​gk​(t)δk​(t)(1)

∫Cg(θ(t))dθ=∫t0t1g(θ(t))⋅θ′(t)dt(2)\int_C\boldsymbol{g}(\boldsymbol{\theta}(t))d\boldsymbol{\theta} =\int_{t_0}^{t_1}\boldsymbol{g}(\boldsymbol{\theta}(t))\cdot\boldsymbol{\theta'}(t)dt\qquad\qquad(2) ∫C​g(θ(t))dθ=∫t0​t1​​g(θ(t))⋅θ′(t)dt(2)

Some Thoughts

整篇文章的核心在 (4) 和 (5) 两个式子，文章说到底就是提出一个新的损失函数，这个损失函数可以根据文章所认为的参数重要性准则来对网络模型的参数做正则化。从(4) 和 (5) 式想到，其实对于联邦学习而言，参数聚合不一定就是client上传模型参数，server加权平均然后再下发参数给client那么粗暴，为了保证聚合参数对client的提升效果，可以在client本地借鉴一下本文的想法，在本地原有模型的基础上向从server端得到的模型进行渐进式的更新。

Paper Note: Overcoming Catastrophic Forgetting in Neural Networks

Created2020-12-07Updated2020-12-16

Overcoming Catastrophic Forgetting in Neural Networks

James Kirkpatrick, Razvan Pascanu, Neil Rabinowitz, Joel Veness, Guillaume Desjardins, Andrei A. Rusu, Kieran Milan, John Quan, Tiago Ramalho, Agnieszka Grabska-Barwinska, Demis Hassabis, Claudia Clopath, Dharshan Kumaran, and Raia Hadsell

Proceedings of the National Academy of Sciences of the United States of America

Paper Adress

Background

人类和动物是可以持续不断地学习新知识，而不会忘了旧知识的。根据相关研究表明，哺乳动物通过将知识保存在新脑皮层（neocortical circuits）中以避免遗忘。当老鼠掌握一项新技能时，一部分兴奋性突触就会增强，这表现为神经元的某些树突的体积增加。重要的是，尽管随后学习了其他任务，这些扩大的树突仍然存在，这说明几个月后老鼠仍然掌握着这些技能。当这些树突被有选择地“擦除”时，相应的技能就会被遗忘。

这些实验发现表明，新脑皮层的持续学习依赖于任务相关的突触巩固。通过使一定比例的突触可塑性降低能够在很长时间范围内稳定持久地编码知识。

Main Idea

论文提出了一种新的持续学习算法 Elastic Weight Consolidation (EWC)，该方法可以看作是在神经网络中模拟了突触巩固。简言之，该方法是把在之前任务中重要的权重的学习率降低——对于之前任务重要的参数变化得慢一点，不重要的参数变化得快一点。

Method

深度神经网络是过度参数化的（overparameterization），也就是说不同参数的神经网络可能有着相同的性能表现。这一理论前提提供了一种可能——适应于任务A的网络 θA∗\theta_A^*θA∗​ 与适应于任务B的网络 θB∗\theta_B^*θB∗​ 可能非常接近。换句话说，如果在持续学习的场景下，如果我们已经找到适应于任务A的网络 θA∗\theta_A^*θA∗​ ，那么我们就很有可能在不太多改变 θA∗\theta_A^*θA∗​ 的情况下找到适应于任务B的网络 θB∗\theta_B^*θB∗​ ，从而使得 θB∗\theta_B^*θB∗​ 在任务A上的表现依然比较好。这种限制以二次惩罚项的形式出现，这样做的好处在于仅仅对范数的限制，可以保证在以 θA∗\theta_A^*θA∗​ 为中心锚点的一定范围内改变某些参数——对之前任务影响大的参数改变少一点，反之改变多一点，所谓的 “elastic” 就体现在这里。

Fig. 1. 在学习完第一个任务 task A 以后得到网络 θA∗\theta_A^*θA∗​。如果我们不加任何限制，直接去学习 task B，那么就是会在 task B上有很好的表现，而“忘记”了 task A，如蓝色箭头所示。如果我们给每个参数都加以相同的约束系数，那么限制可能又太强，而导致为了记住 task A 而学不会 task B，如绿色箭头所示。EWC可以通过计算哪些参数对于task A重要，而哪些参数不重要来灵活地施加约束，从而使得网络既能够学会 task B 也不至忘了 task A，如红色箭头所示。

接下来论文从概率的角度来分析神经网络的训练，从而定义参数相对于某个任务的重要性。从概率的角度来看，当给定一些数据，优化参数就是在找到它们最大概率出现的值。利用贝叶斯公式（Bayes’ rule）：

log⁡p(θ∣D)=log⁡p(D∣θ)+log⁡p(θ)−log⁡p(D)(1)\log p(\theta|\mathcal{D})=\log p(\mathcal{D}|\theta)+\log p(\theta)-\log p(\mathcal{D})\qquad\qquad(1) logp(θ∣D)=logp(D∣θ)+logp(θ)−logp(D)(1)

对数概率log⁡p(D∣θ)\log p(\mathcal{D}|\theta)logp(D∣θ) 其实就是所处理问题的损失函数的相反数，即 −L(θ)-\mathcal{L}(\theta)−L(θ)。如果我们考虑将数据 D\mathcal{D}D 分成两个独立的部分，一部分对应于 task A (DA\mathcal{D}_ADA​) ，另一部分对应于 task B (DB\mathcal{D}_BDB​)。则我们可以重写 (1) 式：

log⁡p(θ∣D)=log⁡p(DB∣θ)+log⁡p(θ∣DA)−log⁡p(DB)(2)\log p(\theta|\mathcal{D})=\log p(\mathcal{D}_B|\theta)+\log p(\theta|\mathcal{D}_A)-\log p(\mathcal{D}_B)\qquad\qquad(2) logp(θ∣D)=logp(DB​∣θ)+logp(θ∣DA​)−logp(DB​)(2)

从上式可以看出，与 task A 相关的项只有 log⁡p(θ∣DA)\log p(\theta|\mathcal{D}_A)logp(θ∣DA​) 。也就是说如果要实现EWC，哪些参数对于 task A 重要的信息必须包括在这一项当中。而这一项本身我们是得不到的，论文使用高斯分布来对这一项做了近似，高斯分布的均值为参数 θA∗\theta_A^*θA∗​ ，diagonal precision 为 the diagonal of the Fisher information matrix F。F 有三点关键属性：(i) 等于损失函数在一个极小值附近的二阶导；(ii) 可以从一阶导计算得到，也就是说即便对于很大的模型来说也是易于计算的；(iii)必为半正定的。

在该近似下，我们在EWC中优化的损失函数 L\mathcal{L}L 为：

L(θ)=LB(θ)+∑iλ2Fi(θi−θA,i∗)2(3)\mathcal{L}(\theta)=\mathcal{L}_B(\theta)+\sum_i\cfrac{\lambda}{2}F_i(\theta_i-\theta^*_{A,i})^2\qquad\qquad(3) L(θ)=LB​(θ)+i∑​2λ​Fi​(θi​−θA,i∗​)2(3)

其中 LB(θ)\mathcal{L}_B(\theta)LB​(θ) 是 task B 的损失函数，λ\lambdaλ 可以理解为旧任务相比于新任务的重要性，iii 标识每个具体参数。

当继续学习第三个任务时，EWC则会继续将网络参数保持在习得 task A 和 task B后的参数附近。

Evaluation

论文对EWC做了一些实验，其中一个是探究EWC是将对不同任务重要的参数都分离开了还是说其实这些任务都是共用了一些相同的参数来做表征。论文通过实验得到两点结论：(1) 各任务间的数据越相似，它们共用的参数越多，Fisher information matrices 的重叠部分（overlap）就越大；(2) 对于任务数据相似性不大的网络而言，随着网络层数加深，共用的参数越多，Fisher information matrices 的重叠部分（overlap）越大。

Discussion

EWC存在两个问题。首先是存储成本问题。EWC需要存储模型之前的参数，而且会随着任务数量的增加而线性增加。除此之外，EWC相当于是要把模型参数固定在第一个任务的经验风险函数的一个local minima附近，那也就是说EWC是探索不到新旧任务之间更好的折衷点——或许在那个折衷点新旧任务都能有更好的表现。

(Sang-Woo Lee, J.-H. Kim, et.al. Overcoming catastrophic forgetting by incremental moment matching. NeurIPS, 2017.) EWC 是用 sequential Bayesian estimation 更新神经网络，从而实现持续学习。也就是说，以一种贝叶斯的方式，用之前任务上训练得到的后验概率来更新新任务的先验概率，而这个先验概率继续用来学习新任务的后验概率。EWC假设后验概率的协方差矩阵是对角阵，即节点之间不相关——尽管这个假设很是脆弱。
